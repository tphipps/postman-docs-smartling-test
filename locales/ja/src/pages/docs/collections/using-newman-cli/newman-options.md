Newman には、実行をカスタマイズするための豊富なオプション セットが用意されています。 オプションのリストを取得するには、フラグを指定して `-h` Newman を実行します。

```bash
$ ニューマンラン-h
```

基本オプション
-------

| オプション             | 細部           |
|:------------------|:-------------|
| `-h`, `--help`    | 使用状況情報を出力する  |
| `-v`, `--version` | バージョン番号を出力する |

セットアップ
------

| オプション                              | 細部                                                           |
|:-----------------------------------|:-------------------------------------------------------------|
| `--フォルダ [フォルダ名]`                   | コレクションから実行するフォルダーを 1 つ指定します。                                 |
| `-e`, `--environment [file\|URL]` | Postman 環境を JSON \[ファイル\] として指定します。                          |
| `-d`, `--iteration-data [file]`    | 使用するデータ ファイル \(JSON または CSV\) を指定します。                      |
| `-g`, `--globals [file]`           | Postman グローバル ファイルを JSON \[ファイル\] として指定します。                  |
| `-n`, `--iteration-count [number]` | 実行する反復回数を定義します。                                              |
| `--working-dir [パス]`               | 相対パスを持つファイルを読み取るときに使用する作業ディレクトリのパスを設定します。 デフォルトは現在のディレクトリです。 |
| `--no-insecure-file-read`          | 作業ディレクトリの外部にあるファイルの読み取りを防止します。                               |
| `--export-environment [パス]`        | Newman が実行を完了する前に最終的な環境変数ファイルを出力するファイルへのパス。                  |
| `--export-globals [パス]`            | Newman が実行を完了する前に最終的なグローバル変数ファイルを出力するファイルへのパス。               |
| `--export-collection [パス]`         | Newman が実行を完了する前に最終的なコレクション ファイルを出力するファイルへのパス。               |

要求オプション
-------

| オプション                  | 細部                                             |
|:-----------------------|:-----------------------------------------------|
| `--delay-request [番号]` | 要求間の遅延 \(ミリ秒単位\) を指定します \[番号\]。              |
| `--timeout [数値]`       | コレクション実行全体の実行が完了するまで待機する時間 \(ミリ秒単位\) を指定します。 |
| `--タイムアウト要求 [番号]`      | 要求の要求タイムアウト \(ミリ秒単位\) を指定します。                |
| `--タイムアウトスクリプト[番号]`    | スクリプトの実行が完了するまで待機する時間 \(ミリ秒単位\) を指定します。      |

その他のオプション
---------

| オプション                               | 細部                                                                                                                                    |
|:------------------------------------|:--------------------------------------------------------------------------------------------------------------------------------------|
| `--保釈`                              | テストケースが失敗したときにランナーを停止します。                                                                                                             |
| `--サイレント`                           | 端末出力をオフにします。                                                                                                                          |
| `●カラーオフ`                            | カラー出力をオフにする \(自動\|オン\|オフ\) \(デフォルト: "自動"\)                                                                                        |
| `--ユニコードを無効にする`                     | ユニコード無効オプションを強制します。 指定すると、出力内のすべてのシンボルが同等のプレーンテキストに置き換えられます。                                                                          |
| `-k`, `--insecure`                  | 厳密な SSL をオフにします。                                                                                                                      |
| `-x`, `--suppress-exit-code`        | 失敗後もテストの実行を続行しますが、 `code=0`                                                                                                           |
| `--リダイレクトを無視する`                     | 回答の自動 `3XX` フォローをオフにします。                                                                                                              |
| `--冗長`                              | コレクションの実行と送信された各要求の詳細情報を表示します。                                                                                                        |
| `--クッキージャー [パス]`                    | JSON クッキー jar のファイル パスを指定します。 ファイルを逆シリアル化するために使用します `tough-cookie` 。                                                                  |
| `--エクスポート-クッキー-jar [パス]`            | Newman が実行を完了する前に最終的な Cookie jar ファイルを出力するファイルへのパス。 ファイルをシリアル化するために使用します `tough-cookie` 。                                             |
| `--グローバル変数 "[グローバル変数名]=[グローバル変数値]"` | コマンドラインでグローバル変数を key=value 形式で指定します。 複数のグローバル変数は、複数回使用して `--global-var` 追加できます。 `--global-var "this=that" --global-var "alpha=beta".` |
| `--env-var "[環境変数名]=[環境変数値]"`       | コマンドラインで環境変数を key=value 形式で設定できます。 複数の環境変数は、複数回使用して `--env-var` 追加できます。 `--env-var "key1=value1" --env-var "key2=value2"`             |

終了ステータス
-------

Newmanは、デフォルトでは、例外なしなど、すべてがうまく実行されると、ステータスコード0で終了します。

継続的インテグレーション \(CI\) ツールは、これらの終了コードに応答し、それに応じてビルドの合格または失敗をします。

or `--suppress-exit-code` を使用して `-x` 、現在の実行の既定の終了コードをオーバーライドできます。

この `--bail` フラグを使用して、状態コード 1 のテスト ケース エラーで停止するように Newman に指示し、CI ツールまたはビルド システムで取得できます。

```bash
$ newman run PostmanCollection.json -e environment.json --bail
```

データ ファイルの例
----------

反復ごとの変数など、異なるデータ セットを提供するには、フラグを使用して `-d` JSON ファイルまたは CSV ファイルを指定します。

たとえば、次のようなデータ ファイルでは *2 つの* 反復が実行され、各反復では一連の変数が使用されます。

```json
[{
    "url": "http://127.0.0.1:5000",
    "user_id": "1",
    "id": "1",
    "token_id": "123123",
},
{
    "url": "http://postman-echo.com",
    "user_id": "2",
    "id": "2",
    "token_id": "899899",
}]
```

```bash
$ newman run mycollection.json -d data.json
```

この変数セットの CSV ファイルの例を次に示します。

```bash
URL、user_id、ID、token_id
http://127.0.0.1:5000、1、1、123123123
http://postman-echo.com、2、2、899899
```
